# Falkor Self-Deploy Capability

## What
Falkor (asgard-ai v6.5.0) can now patch and deploy any Cloudflare Worker autonomously via `POST /admin/deploy`. It can also read source via `GET /admin/worker-source?name=X` and inspect bindings via `GET /admin/worker-settings?name=X`.

## Verified
2026-05-13 — first Falkor-initiated deploy was to falkor-brain (deployment_id `cd20f146d44a49579c0bea9d61e4a048`). The stamp comment was appended and reproduced.

## Mechanism
- Pulls existing worker bindings from `/versions/{latest_id}/resources.bindings` (the `/settings` endpoint omits newer types like Vectorize/AI).
- Rebuilds metadata with `inherit` for secret_text bindings so values are preserved.
- Multipart PUT to CF Workers API with `Content-Type: application/javascript+module` and `filename=index.js` (the filename trick is required — a temp filename gives a 10021 `No such module: index.js` error).
